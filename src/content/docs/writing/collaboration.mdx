## 多模型协作写作

### tmux 工作流配置

让 Claude / GPT-5.2 / Gemini Pro 分工协作：

| 实例 | 负责内容 | 协作方式 |
|------|---------|---------|
| Instance 1 | Related Work | 文献梳理与对比 |
| Instance 2 | Method 写作 | 方法描述与形式化 |
| Instance 3 | 实验描述与表格 | 实验设计和结果 |
| Instance 4 | 一致性校对 | 术语、符号、变量统一 |

### 关键原则

- **同时进行，而非顺序**
- **定期同步，避免冲突**
- **共享术语表，确保一致性**

## 术语表管理

### 术语表示例

```markdown
# 术语表

| 中文 | 英文 | 缩写 | 定义 |
|------|------|------|------|
| 注意力机制 | Attention Mechanism | Attn | ... |
| 多头注意力 | Multi-Head Attention | MHA | ... |
| 自注意力 | Self-Attention | SA | ... |
```

### 使用建议

1. **统一维护**: 单一文件管理所有术语
2. **版本控制**: 使用 Git 追踪变更
3. **定期同步**: 各实例定期更新术语表
4. **冲突解决**: 建立术语决策流程

## 协作工作流

### 阶段 1: 大纲生成

使用 Claude Opus 4.6 生成长大纲（200K 上下文）

### 阶段 2: 分段写作

各实例并行处理不同章节

### 阶段 3: 一致性检查

使用专门实例校对一致性

### 阶段 4: 整合优化

由最强模型整合所有内容

## 常见写作误区

- ❌ "像论文但没信息" - 缺乏实质贡献
- ❌ 过度声明 - claim 超出实验支持
- ❌ 忽略关键 baseline - 对比不充分
- ❌ 叙事不清 - 逻辑跳跃
