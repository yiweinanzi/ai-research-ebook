## 完整研究工作流

从 idea 到论文发表的典型时间线（10-14周）：

---

## Week 1-2: 调研阶段

### 目标
产出结构化的研究地图和空白清单

### 工作流

```
Elicit 快速扫描领域
    ↓
Sci-Reasoning 分析创新模式
    ↓
产出 Research Gap 清单
```

### 具体任务

1. **使用 Elicit** 扫描领域关键词
2. **使用 Semantic Scholar** 探索研究脉络
3. **使用 Sci-Reasoning** 分析创新模式
4. **整理研究地图**：
   - 问题子类分类
   - 方法范式分类
   - 评测基准整理

### 产出

- Research Gap 清单（10-20个候选）
- Benchmark & Metric Sheet
- Assumption Matrix

---

## Week 3: 项目设计

### 目标
确定研究问题和实验计划

### 工作流

```
Claude Code 生成项目文档
    ↓
GPT-5.2 审查 claim-实验对齐
    ↓
确定实验计划
```

### 具体任务

1. **生成项目文档**（使用 Claude Code）
2. **审查文档**：
   - claim-实验是否对齐？
   - baseline 是否足够强？
   - 是否有致命假设？
3. **确定实验计划**：
   - 主实验设计
   - 消融实验清单
   - 压力测试场景

### 产出

- 项目文档（20-30页）
- 实验计划
- 里程碑时间表

---

## Week 4-7: 代码实现 + 实验

### 目标
完成实现和主要实验

### 工作流

```
Claude Code + Sub-agents 并行开发
    ↓
自动化 ablation/stress test
    ↓
实验结果自动汇总
```

### 具体任务

1. **并行开发**（使用 Claude Code）：
   - Architect ×1: 总体结构
   - Implementer ×2: 核心模块
   - Tester ×1: 测试和评测

2. **实验执行**：
   - 主实验：与 baseline 对比
   - 消融实验：每个模块的贡献
   - 压力测试：边界条件

3. **结果汇总**：
   - 自动生成主结果表
   - 消融实验图表
   - 失败案例分析

### 产出

- 可复现的代码库
- 实验结果（主表 + 消融 + 分析）
- 复现指南

---

## Week 8-9: 论文写作

### 目标
完成论文初稿

### 工作流

```
Prism + GPT-5.2 生成初稿
    ↓
多模型协作完善各章节
    ↓
PaperBanana 生成图表
```

### 具体任务

1. **生成大纲**（使用 Claude Opus 4.6）
2. **并行写作**（多实例）：
   - Instance 1: Related Work
   - Instance 2: Method
   - Instance 3: Experiments
   - Instance 4: Introduction + Conclusion

3. **图表制作**：
   - 使用 PaperBanana 生成草图
   - Figma 精细化
   - 整合到论文

### 产出

- 论文初稿（8-10页）
- 所有图表
- 补充材料

---

## Week 10: 审稿迭代

### 目标
通过多轮审稿迭代提升论文质量

### 工作流

```
10+ 轮自动审稿
    ↓
TODO 驱动的修改
    ↓
人工最终审查
```

### 具体任务

1. **多模型并行审稿**：
   - Claude Opus 4.6
   - GPT-5.2 Pro
   - Gemini Pro

2. **交叉验证意见**：
   - 找出真正的问题
   - 按优先级排序

3. **批量修改**：
   - 每轮处理3-5条
   - 确保验收标准

4. **验证修复**：
   - 重新审稿
   - 确认问题解决

### 产出

- 稳定的投稿版本
- 审稿意见记录
- 修改说明

---

## Week 11: 投稿

### 目标
完成投稿准备

### 具体任务

1. **格式检查**
2. **补充材料准备**
3. **提交**

---

## Week 12+: Rebuttal

### 目标
回应审稿意见

### 工作流

```
分析审稿意见
    ↓
分类回应
    ↓
补充实验
```

### 具体任务

1. **分析审稿意见**：
   - 可修复的
   - 误解的
   - 不同意但合理的
   - 不同意的

2. **准备回应**：
   - 使用 Rebuttal 模板
   - 准备补充实验
   - 标注修改位置

### 产出

- Rebuttal 文档
- 修改后的论文
- 补充实验结果
